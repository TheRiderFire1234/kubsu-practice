## Генетический алгоритм

### Кодирование хромосом
- **Требование ТЗ:** Бинарное кодирование
- **Реализация:** 3-битные строки в классе Individual
- **Статус:** ✅

### Стратегия создания начальной популяции
- **Требование ТЗ:** 
  - A. Стратегия «одеяла» (равномерное распределение)
  - C. Стратегия фокусировки (около 1-2)
- **Реализация:** `createInitialPopulation()` (строка 66)
  - strategy == 1: Одеяло
  - strategy == 2: Фокусировка
- **Статус:** ✅

### Вид селекции
- **Требование ТЗ:**
  - A. Случайная селекция
  - B. Инбридинг (из лучшей половины)
- **Реализация:** `selectParent()` (строка 87)
  - selectionType == 1: Случайная
  - selectionType == 2: Инбридинг
- **Статус:** ✅

### Оператор кроссинговера
- **Требование ТЗ:**
  1. Стандартный одноточечный
  2. Стандартный двухточечный (многоточечный)
  3. Частично соответствующий одноточечный
  4. На основе чисел Фибоначчи
- **Реализация:** `chooseCrossover()` (строка 190)
  - crossoverType == 1: Одноточечный
  - crossoverType == 2: Двухточечный
  - crossoverType == 3: Частично соответствующий одноточечный
  - crossoverType == 4: Фибоначчи
- **Статус:** ✅

### Операторы мутации и инверсии
- **Требование ТЗ:**
  1. Точечная мутация
  2. Транспозиция (инверсия)
- **Реализация:** `chooseMutation()` (строка 197)
  - mutationType == 1: Точечная мутация
  - mutationType == 2: Транспозиция
- **Статус:** ✅

### Оператор отбора
- **Требование ТЗ:** Пропорциональный отбор
- **Реализация:** `proportionalSelection()` (строка 172)
  - Использует вес, обратный функции приспособленности
  - `weights.push_back(1.0 / (1.0 + ind.fitness))`
- **Статус:** ✅

### Схема эволюции

  - 2 потомка за поколение
  - Независимые кроссовер и мутация для каждого потомка
  - Пропорциональный отбор для сохранения поколения
- **Статус:** ✅

## Графическое отображение

### Требование ТЗ
- Красная точка: найденный оптимум
- Синие точки: промежуточные значения на каждом шаге
- Черная линия: график функции f(x)
- Инструмент: gnuplot

### Реализация
Функция `writePlotFiles()` (строка 209) генерирует:
1. `func_data.txt` - значения функции для всех x
2. `trace_data.txt` - лучшие значения каждого поколения
3. `best_point.txt` - найденный оптимум
4. `plot.gnuplot` - скрипт для построения графика:
   - Черная линия: данные из func_data.txt
   - Синие точки: данные из trace_data.txt
   - Красные точки: данные из best_point.txt

**Статус:** ✅
